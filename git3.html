git으로 협업하기

일반 사용자는 public repository의 클론은 가능하지만 Push는 거부된다.

$git clone < 원격저장소 주소 > [클론할 저장소 폴더]

public repository의 원격 저장소에서 협업자로 초대해야 가능하다.

Collaborator가 초대장 내용을 확인하고 초대를 수락하면 Commit을 Push하고 URL 없이도 클론이 가능한 권한이 생성된다.

참여자의 파일 업로드 확인

  • 새로운 파일 생성
  • $vi first_uploard.txt

  • 위에서 작성한 파일을 스테이징과 커밋 수행
  • $git add first_upload.txt
  • $git commit -m "first commit"

  • 원격 저장소로 푸시
  • $git push origin master

주의 사항

관리자의 허가없이 master branch에 파일을 올리지 말 것

Master branch는 배포를 위한 최종 제품에 해당한다. 아무나 업데이트하면 심각한 문제가 될 수 있다.

Branch 규칙 지정

master merge 전에 pull request 기능을 통해 관리자나 팀원들의 동의를 얻는다.

기대 효과: master branch의 보호, 코드 리뷰를 통한 학습, 코드 품질 향상을 도모할 수 있다.

저장소 화면의 Settings -> Branches -> add rule 메뉴 선택

안전한 협업을 위해 지켜야 할 사항

  • 항상 브랜치를 사용하여 파일을 업로드 후, 관리자의 승인을 구한다.
  • 승인이 되면 master branch에 변경 사항을 병합한다.
  • 참여자는 push origin [브랜치 이름] -> Create Pull Request 클릭
  • Reviewer 선택
  • 관리자는 Pull request 확인 (Pull request 탭클릭) > 코드 변경 사항 확인 > Review change에서 코멘트, 승인, 변경요청 > 승인

참여자에 의한 브랜치 생성과 파일 업로드

  • 로컬 저장소 브랜치 생성
  • $git checkout -b (branch name)

  • 생성한 브랜치와 현재 위치 확인
  • $git branch

  • 현재 브랜치를 원격 저장소에도 만들고 싶을 떄
  • git push origin (current_branch_name)

  • 원격 저장소에 만들어진 브랜치 확인
  • $git branch -r (or) $git branch -r -v

  • 로컬 브랜치를 원격 저장소의 브랜치와 연동
  • git branch --set-upstream-to origin/(branch name)

  • 로컬 브랜치에서 소스코드 업데이트
  • $git first_upload.txt
  • #staging과 commit 수행#
  • $git add (filename)
  • $git commit -m "commit"

  • 로컬 브랜치에서 커밋한 코드 업로드
  • $git push origin (branch name)

Pull request 하기

참여자 게정으로 로그인 후, code > Compare & pull resquest

안전한 협업을 위해 지켜야 할 사항

Pull Request 메시지 작성과 요청

관리자는 Pull Request 요청 일람을 확인

관리자는 Files changes > Review changes 항목을 클릭하여 코멘트, 변경 요청, 승인할 수 있다.

최종 검토가 끝나면 Merge Pull request와 Confirm merge 버튼을 클릭하여 master branch로 병합

master branch 병합 후

작업 브랜치는 삭제한다.

또 다른 작업이 필요하면 다시 브랜치를 생성해서 작업한다.

브랜치 삭제

$git checkout branch2

$git branch --delete branch1 #로컬 브랜치 삭제

$git push origin :dev1

Commit한 내용을 Master branch에서 확인
