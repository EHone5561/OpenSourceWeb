1.마크다운 Markdown

1.1.마크다운이란?

Markdown은 텍스트 기반의 마크업 언어이다.
2004년 존그루버에 의해 만들어졌으며 쉽게 쓰고 읽을 수 있으며 HTML로 변환이 가능하다.
특수기호와 문자를 이용한 매우 간단한 구조의 문법을 사용하여 웹에서도 보다 빠르게 컨텐츠를 작성하고 보다 직관적으로 인식할 수 있다.
마크다운이 최근 각광받기 시작한 이유는 깃헙(https://github.com) 덕분이다.
깃헙의 저장소 Repository에 관한 정보를 기록하는 README.md는 깃헙을 사용하는 사람이라면 누구나 가장 먼저 접하게 되는 마크다운 문서다.
마크다운을 통해서 설치방법, 소스코드 설명, 이슈 등을 간단하게 기록하고 가독성을 높일 수 있다는 강점이 부각되면서 점점 여러 곳으로 퍼져가게 되었다.

1.2.마크다운의 장단점

1.2.1. 마크다운의 장점

  1. 간결하다.
  2. 별도의 도구없이 작성가능하다.
  3. 다양한 형태로 변환이 가능하다.
  4. 텍스트(Text)로 저장되기 때문에 용량이 적어 보관이 용이하다.
  5. 텍스트파일이기 때문에 버전관리시스템을 이용하여 변경이력을 관리할 수 있다.
  6. 지원하는 프로그램과 플랫폼이 다양하다.

1.2.2. 마크다운의 단점

  1. 표준이 없다.
  2. 표준이 없기 때문에 도구에 따라서 변환방식이나 생성물이 다르다.
  3. 모든 HTML 마크업을 대신하지 못한다.

3.2.마크다운을 편하게 작성하기 위한 에디터

VS Code Plugin

MarkDown은 개발자들이 글을 적을때 많이 사용하는 마크업 언어입니다.
그리고 VS Code는 개발자들이 많이 사용하는 에디터이자 IDE입니다.
실제로 VS Code에 MarkDown 형식의 파일을 작성할 수 있지만, 실질적인 도움은 주지 않습니다.
하지만 VS Code는 플러그인을 활용할 수 있죠.
아래의 링크를 통해 VS Code에서 MarkDown을 쉽게 작성할 수 있도록 도와주는 VS Code 플러그인을 볼 수 있습니다.
VS Code Markdown Plugin
VS Code Markdown Plugin Github

Typora

typora는 PC의 모든 운영체제에서 사용하는 무료 MarkDown 에디터입니다.
편하고 쉽게 일반 글을 쓸 수 있도록 잘 만들어져 있습니다.
typora를 사용하면 다른 프로그램으로 markdown을 작성하고 싶지 않게 됩니다.
그리고 작성하기 어려운 테이블도 작성하기 쉽게 만들어주고, 다른 문법도 클릭 하나로 만들 수 있게 해줍니다.
Typora

Markdown 테이블 제네레이터

MarkDown에서 테이블을 작성할 수 있지만, 하나하나 타이핑하여 테이블을 작성하는건 아주 힘듭니다.
물론 typora를 사용하면 표를 작성하는것은 편하지만, 이외에 다른 방법으로 작성할 때 어려움을 느낄 수 있습니다.
이럴때 테이블 제너레이터 사이트로 들어가서 쉽게 테이블을 작성할 수 있습니다.
Markdown 테이블 제네레이터

Notion

Notion은 실제로 MarkDown 에디터라고 하기 어렵지만, Notion으로 작성한 글들을 MarkDown으로 추출할 수 있기 때문에 포함했습니다.
Notion은 다양한 형식의 템플릿과, 메모, 협업, 프로젝트 관리 등등 기능이 많아 유용한 프로그램입니다.
